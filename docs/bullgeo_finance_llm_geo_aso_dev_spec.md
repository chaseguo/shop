# BullGEO 金融信贷 GEO+ASO 平台开发文档（v2.0）

> 面向：Cursor 开发团队 / 产品 / 增长 / 合规 / 内容运营  
> 站点：`bullgeo.com`  
> 版本目标：将“LLM 检索可见度 + 合规信任度 + ASO 承接效率”做成可监控、可诊断、可修复系统。

---

## 1. 项目背景与目标

基于 2026 年 2 月内部抽样（真人与群控同口径问法）结果：

- 奇富借条推荐占比 40%，显著高于行业均值 23%。
- 洋钱罐在多平台表现均衡，负面占比较低。
- 度小满推荐占比 18%，低于行业均值，且“被看见 + 被信任”双弱。

**结论**：GEO 是上游入口，ASO 是承接与沉淀；必须联动，才能把“可见度”沉淀为“长期权重资产”。

## 2. 2026 金融信贷行业 LLM 检索底层规则（全平台通用）

> 本章节为平台策略最高优先级约束，所有算法、内容生产、分发策略必须遵循。

### 2.1 通用规则

1. **强监管行业信源权重排序**：
   监管机构官方发布 > 持牌金融机构官方内容 > 权威垂直财经媒体 > 合规机构 UGC > 个人自媒体。  
   （无资质个人内容在金融问答几乎无引用资格）
2. **合规过滤阈值**：
   无资质、虚假宣传、违规承诺类内容进入“强过滤”，不进入检索引用池。
3. **同主体同口径多渠道一致性**：
   同主体、同口径、同证据链内容，会被判定为高可信权威内容，引用权重显著提升。

### 2.2 平台策略设计原则（产品内强制）

- 每个内容条目必须有：`证据来源`、`资质类型`、`版本号`、`更新时间`、`合规状态`。
- 每个分发动作必须记录：`渠道`、`主体账号`、`发布时间`、`回链URL`、`收录状态`。
- 每次监测报告必须区分：`可见度问题` 与 `信源资质问题`。

## 3. 四大平台 + 腾讯元宝（混元）检索源策略（2026.02）

## 3.1 字节跳动・豆包

- **第一优先级（约 75%）**：
  字节自有生态（头条号/抖音蓝V/抖音百科/抖音财经专栏/头条搜索品牌专区）+ 监管官方源。
- **第二优先级（约 20%）**：
  知乎机构号、持牌机构官网/白皮书/公告、东方财富/同花顺/雪球机构内容。
- **第三优先级（≤5%）**：
  合规机构自媒体（百家号/搜狐号等）。
- **专属规则**：字节生态白名单优先级极高，时效和抓取优先级领先。

## 3.2 月之暗面・Kimi

- **第一优先级（约 72%）**：
  知乎机构号 + 权威财经媒体（财新/一财/21世纪/证券时报等）+ 监管/持牌官方内容 + 行业研究报告。
- **第二优先级（约 22%）**：
  消费场景类机构UGC、垂直金融平台机构深度内容。
- **第三优先级（≤6%）**：
  合规机构自媒体与门户财经频道。
- **专属规则**：长文本、结构化、可溯源内容权重更高。

## 3.3 腾讯元宝（底层：混元 + 微信/搜狗检索体系）

- **第一优先级（约 78%）**：
  微信生态核心源（公众号/视频号蓝V/小程序说明页/搜一搜品牌专区）+ 腾讯财经体系 + 监管/持牌官方内容。
- **第二优先级（约 17%）**：
  知乎机构号、搜狗高权重财经站点。
- **第三优先级（≤5%）**：
  合规机构自媒体、综合门户财经频道。
- **专属规则**：微信生态具最高白名单权重，竞品生态内容权重明显更低。

## 3.4 DeepSeek

- **第一优先级（约 80%）**：
  国家级监管源 + 持牌机构官方源 + 权威财经媒体 + 头部研究机构报告。
- **第二优先级（约 15%）**：
  知乎机构号、官方合规行业站点。
- **第三优先级（≤5%）**：
  合规机构自媒体、门户财经官方频道。
- **专属规则**：权威性门槛最高，无资质无背书内容基本 100% 过滤。

## 4. 产品定位：Detect / Diagnose / Structure / Evidence Wiki / ASO

1. **Detect（可见度雷达）**：追踪豆包/Kimi/腾讯元宝/DeepSeek 推荐率与Top位占比。
2. **Diagnose（负面主题拆解）**：识别利率误解、资费不清、审批误读、品牌混淆等。
3. **Structure（AI友好结构化）**：把品牌优势改写为可引用 FAQ/条款解释/对比条目。
4. **Evidence Wiki（证据化资产）**：沉淀“有证据、可追溯、可复用”条目库。
5. **ASO Bridge（承接）**：把 GEO 主题映射到应用商店关键词、文案、截图与评价引导。

---

## 5. 指标体系（新增平台级可信度指标）

### 5.1 北极星指标

`LRCI = 0.30*推荐率 + 0.20*Top1占比 + 0.20*正向提及率 - 0.15*负面提及率 + 0.15*可信信源占比`

### 5.2 关键业务指标

- 推荐率 / Top1 / Top3（按平台）
- 负面主题占比（按主题 + 严重度）
- 可信信源占比（监管/持牌/权媒来源占比）
- 同口径跨渠道一致性得分
- GEO -> ASO 承接率

## 7. Logo / Icon 直观化实现规范（你要求的可视化）

## 7.1 实施策略

- 页面允许两种模式：
  1) **官方品牌 Logo（优先）**：需法务确认品牌使用授权。
  2) **中性平台 Icon（默认）**：使用本项目内置 SVG，规避商标授权风险。

## 7.2 代码落地要求

- 资源目录：`public/icons/models/`
- 文件命名：`doubao.svg` / `kimi.svg` / `yuanbao.svg` / `deepseek.svg`
- 前端组件：`ModelSourceBadge`（名称 + icon + 权重 + 策略说明）

## 7.3 UI 规格

- icon 尺寸：24 / 32 / 40 px 三档。
- 深色背景提供描边版本。
- 所有 icon 需有 `aria-label` 与 `title`，便于可访问性与 SEO。

## 9. API 草案（新增平台与信源接口）

- `GET /api/v1/platforms`：返回支持平台（含腾讯元宝）。
- `POST /api/v1/detect/jobs`：创建检测任务（必须带平台）。
- `GET /api/v1/reports/source-authority`：可信信源覆盖报告。
- `GET /api/v1/reports/platform-radar`：平台偏好雷达图数据。
- `POST /api/v1/assets/{id}/consistency-publish`：同口径多渠道发布。

---

## 10. 合规与风控（金融信贷必选）

1. 所有“利率/资费/额度”必须带边界条件。
2. 禁止“100%通过”“绝对最低”等承诺表达。
3. 每条对比信息必须有来源 + 时间戳。
4. 素材发布前必须经过合规审核节点。
5. Logo 使用遵循授权优先策略，未授权时仅用中性 icon。

## 13. 90天成功定义

1. 度小满推荐率相对基线提升 >= 30%。
2. “利率误解/资费不清”负面主题占比下降 >= 25%。
3. 可信信源占比提升 >= 35%。
4. 四平台（含腾讯元宝）均形成可持续的内容资产引用闭环。
